WF4-KPI-001: KPI Pack auto-loaded from WF2/WF3 registry. The Pack header fields (Family, Domain, Technique, Data Type, Data Structure, Model Count, Version, Published Date) are non-editable display fields. The model name is shown alongside the Pack ID for context.
WF4-KPI-002: "Is Primary" is a per-row independent toggle (⭐). Multiple metrics can be marked Primary simultaneously across all sections. Selecting primary unlocks Warning, Breach, and applicable Delta threshold inputs for that row. Non-primary rows remain locked. Direction is always non-editable regardless of primary state.
WF4-KPI-003: Concept Drift absolute threshold fields are permanently "Not applicable" — this cannot be changed. The Delta applicable toggle is a read-only display inherited from WF3. Metrics showing Not set have no threshold configured in WF3 — they are computed but no RAG status is returned until thresholds are configured in WF3.
WF4-KPI-004: Include checkbox deselection excludes a metric from this ingestion run's computation only — the WF3 KPI Pack is unaffected. Deselected rows disable all inputs including the primary toggle.
WF4-KPI-005: Threshold validation fires on input: for "Lower is Better" metrics, Warning threshold must be less than Breach threshold; for "Higher is Better" metrics, Warning must exceed Breach. An invalid entry shows a red border and inline error. The RAG Trigger Rule column updates live to reflect current threshold values.
