WF3.1-KPI-001: This wireframe represents the KPI Review & Configuration step within Model Data Ingestion. The KPI Pack is auto-tagged from the WF2 registry to MDL-001 based on its Model Family. All 23 metrics are pre-loaded with WF2-published thresholds.

WF3.1-KPI-002 (Primary Toggle): Clicking ⭐ on any row marks it as Primary — the row turns yellow, the Mode selector unlocks, and all applicable threshold fields for that row become editable. Multiple rows can be primary simultaneously. Non-primary rows remain read-only; direction is always non-editable.

WF3.1-KPI-003 (Mode Selector): Available only for Primary metrics. Absolute — monitoring fires when the metric crosses the absolute Warning/Breach value. Delta — monitoring fires when the metric changes by ΔWarning/ΔBreach from the reference baseline. Both — both absolute and delta thresholds are active simultaneously; alerts fire when either condition is triggered. Switching mode activates/deactivates the respective threshold columns for that row.

WF3.1-KPI-004 (Validation): For Higher is Better metrics: Warning must be > Breach (both absolute and delta). For Lower is Better metrics: Warning must be < Breach. Concept Drift metrics (IV, FI, SHAP) have no direction — delta thresholds are treated as magnitude thresholds (both positive and negative shifts). Invalid entries show red borders and inline error messages.

WF3.1-KPI-005 (Not Set): Feature Importance and SHAP had no thresholds configured in WF2. When a user marks these as Primary and enters thresholds here, they are stored as run-specific overrides and do not modify the WF2 pack. Fields marked "Not set" are computed without RAG status until thresholds are entered.

WF3.1-KPI-006 (Scope): Changes confirmed here apply to this ingestion run only (MDL-001). They do not modify the published WF2 KPI Pack. To change the pack permanently, use WF2 KPI Configuration → Edit Pack.
